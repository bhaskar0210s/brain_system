# Consolidation Workflow (DPMA)

This document describes the recommended process for performing consolidation on brain-dot-md.

## When to Run Consolidation
- After significant new work or learning
- When context feels fragmented or retrieval is becoming inefficient
- Periodically (e.g., end of week or major milestone)
- When explicitly requested

## Step-by-Step Process

### Phase 1: Review & Capture
1. Review recent activity, new decisions, and any memorization_inbox items.
2. Identify raw material that has lasting value.

### Phase 2: Classification
For each significant item, decide:
- **L1 Candidate?** → Run full Change Proposal process (very conservative).
- **L2 Material?** → Integrate into semantic architecture (create/update files, abstractions, patterns).
- **L3 Material?** → Keep in episodic memory or archive later.
- **Low value?** → Archive or discard.

### Phase 3: Organization (L2 Focus)
- Create new files/folders in L2 if current structure feels insufficient.
- Refactor or merge related content.
- Update `current_mental_model.md` and `key_abstractions.md`.
- Extract patterns into `patterns_and_anti_patterns.md`.

### Phase 4: Update Importance & Relationships (L4)
- Adjust importance scores in `master_importance_index.md`.
- Add or update relationships in `relationship_graph.md`.
- Refine `retrieval_priorities.md` if prompt patterns have changed.

### Phase 5: Meta-Reflection (L5)
- Record what was done in `consolidation_history.md`.
- Note any observations about the system's own effectiveness in `self_optimization_notes.md`.

### Phase 6: Documentation
- Briefly summarize changes made during this consolidation pass.
- Ensure future retrieval will benefit from the updates.

## Principles
- Be **generous in L2 creation**, **extremely conservative in L1 changes**.
- Prioritize **retrieval quality** over perfect organization.
- Use **importance scoring and linking** as the main tools for plasticity.

*This workflow should be followed (and can be improved) by the LLM during each consolidation pass.*
